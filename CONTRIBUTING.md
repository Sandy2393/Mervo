# Contributing to Mervo

Thank you for contributing! Please follow these guidelines to keep the project healthy.

## Code style
- Use TypeScript + Prettier formatting
- Follow the existing architecture (services, types.ts, pages, lib)

## Branches & PRs
- Use feature branches and open PRs to `main` for review
- Include a short description and link to relevant issue
- Add unit tests for new functionality

## Commit messages
- Use conventional commits (feat:, fix:, chore:, docs:)

## Security
- Do not commit secrets; use `.env.example` as the template

## Testing
- Run unit tests and lint before opening PR

## Release
- Update `RELEASE_NOTES_TEMPLATE.md` and version accordingly
